# 电商管理后台 - 信息架构需求

## 导航结构

一级导航：总览、订单、商品、库存、用户、运营数据、权限设置。

桌面端使用常驻侧边导航和顶部工具栏；移动端使用抽屉式导航，并优先展示当前页面标题、搜索和主要筛选。

## 信息对象

- Order：订单号、用户、金额、状态、支付状态、履约状态、创建时间、异常标签。
- Product：商品 ID、名称、SPU/SKU、类目、价格、状态、图片、库存摘要。
- Inventory：SKU、仓库、可售库存、锁定库存、安全库存、更新时间。
- InventoryBatch：批次号、SKU、数量、生产日期、有效期、剩余天数、预警状态、权限范围。
- User：用户 ID、昵称、等级、联系方式脱敏值、订单数、最近购买时间。
- OperationMetric：GMV、订单数、转化率、客单价、退款率、时间范围、刷新时间。
- Permission：角色、资源、动作、字段级可见性。

## 页面层级

总览页面以 KPI 和异常摘要为入口；列表页面以筛选工具栏、表格、分页和详情抽屉为主体；库存页面需要在 SKU 行内展示批次预警摘要，并允许进入批次明细。

## 内容优先级

高优先级：异常状态、金额/库存/有效期、订单和 SKU 标识、权限反馈。

中优先级：趋势图、细分指标、用户等级、商品类目。

低优先级：内部备注、历史操作记录、非关键装饰信息。
