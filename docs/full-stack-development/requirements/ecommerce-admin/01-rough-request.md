# 电商管理后台 - 粗需求澄清

## 原始诉求

用户只给出一句话：“我要一个电商管理后台，可以看订单、商品、库存、用户和运营数据。”

## Agent draft

这是一个面向运营、客服、仓储和管理员的内部电商管理后台。第一版应以“看得见、查得快、能定位异常”为核心，不默认进入下单、财务结算、复杂营销配置或仓储作业系统。

后台至少包含总览、订单、商品、库存、用户、运营数据六个一级页面。每个页面必须支持列表浏览、筛选、搜索、状态识别、详情进入和异常反馈。库存页面在模拟变更后增加“批次有效期预警”和“权限控制”。

## 范围边界

范围内：订单查询、商品管理视图、库存视图、用户视图、运营指标、基础 RBAC、批次有效期预警、桌面和移动端查看体验。

范围外：支付清结算、完整 WMS、客服工单系统、营销活动搭建器、真实生产数据迁移、第三方平台对接。

## Assumptions to verify

- 默认这是一个 React Web 管理后台，实际项目目录当前未发现已有前端工程。
- 默认第一版偏“管理和分析”，写操作只保留必要的商品编辑、库存预警规则配置和详情抽屉。
- 默认权限至少分为管理员、运营、客服、仓储四类。

## Questions for user confirmation

- 这套后台第一版是否以“读多写少”的管理视图为主？
- 是否接受管理员、运营、客服、仓储四类角色作为初始权限模型？
- 库存有效期预警是否按批次维度展示，并允许按临期天数筛选？
