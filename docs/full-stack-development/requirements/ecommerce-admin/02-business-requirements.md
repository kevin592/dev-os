# 电商管理后台 - 业务需求

## Agent draft

电商管理后台的业务目标是提升运营可见性和异常处理效率。运营人员需要快速判断订单、商品、库存、用户和数据表现是否健康；仓储人员需要识别库存不足、超卖风险和批次临期风险；管理员需要控制访问权限，避免越权查看或修改敏感信息。

## 业务目标

- 订单问题定位：在 30 秒内通过筛选和搜索定位异常订单。
- 商品经营检查：快速识别下架、低库存、缺图、价格异常等商品。
- 库存风险治理：展示库存不足、锁定库存、批次临期和过期风险。
- 用户运营分析：查看新增、复购、沉默用户和等级分布。
- 经营数据复盘：按时间范围查看 GMV、订单量、转化、客单价、退款率。

## 业务规则

- 敏感用户信息默认脱敏，只有管理员或授权客服可查看完整信息。
- 库存批次有效期按 `expiresAt` 和预警阈值计算状态：正常、临期、过期。
- 缺少权限时必须显示可理解的禁用态或权限提示，不允许静默失败。
- 运营数据必须显示时间范围、刷新时间和数据口径。

## 风险

- 若没有后端权限契约，前端容易出现“看似隐藏、实际可调接口”的越权风险。
- 若没有状态矩阵，表格在空数据、接口错误和移动端窄屏下容易失真。
- 若直接照搬 TailwindUI/Catalyst 源码，会突破 HeroUI 官方组件优先边界。
