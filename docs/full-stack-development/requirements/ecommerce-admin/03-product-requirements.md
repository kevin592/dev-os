# 电商管理后台 - 产品需求

## 用户角色

- 管理员：查看全部模块，配置权限，查看完整用户和库存批次信息。
- 运营：查看订单、商品、用户和运营数据，可编辑商品基础运营字段。
- 客服：查看订单与用户详情，敏感字段按权限脱敏。
- 仓储：查看库存、批次和预警，不能访问用户敏感信息。

## 页面范围

- 总览：核心 KPI、订单趋势、库存风险、运营提醒。
- 订单：订单表格、状态筛选、搜索、详情抽屉、异常标记。
- 商品：商品表格、上下架状态、库存摘要、编辑入口。
- 库存：SKU 库存、锁定库存、可售库存、批次有效期、临期/过期预警、权限状态。
- 用户：用户列表、等级、最近购买、脱敏联系方式、详情。
- 运营数据：GMV、订单量、转化率、客单价、退款率、时间范围筛选。

## 模拟变更纳入

变更请求：“库存页面需要增加批次有效期预警和权限控制。”

产品层新增要求：

- 库存页必须支持按批次查看 `batchNo`、`expiresAt`、`remainingDays`、`expiryStatus`。
- 预警状态至少包括正常、临期、过期；临期阈值可配置，默认 30 天。
- 无权限用户能看到库存汇总，但不能查看批次成本、供应商、调整动作。
- 缺权限的操作入口必须显示禁用态和原因，不能直接隐藏所有上下文。

## 非目标

- 不做完整采购入库、盘点、调拨、财务核算。
- 不做营销活动搭建器。
- 不做真实第三方平台同步。
