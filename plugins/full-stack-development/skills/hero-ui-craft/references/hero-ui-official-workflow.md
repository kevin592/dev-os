# HeroUI Official Workflow

Use this reference when a front-end task needs HeroUI React v3 setup, migration, or quality review.

## Official Context

- Read `references/official/react/AGENTS.md` first. It is generated by the official `heroui-cli agents-md --react --output AGENTS.md` command and is the priority agent entry for this plugin.
- Use `references/official/react/.heroui-docs/react/` for bundled component, getting-started, release, demo, and pattern files referenced by `AGENTS.md`.
- Download project-local AI docs with `npx heroui-cli@latest agents-md --react --output AGENTS.md`.
- Use `https://heroui.com/react/llms.txt` for quick reference.
- Use `https://heroui.com/react/llms-components.txt` for component documentation.
- Use `https://heroui.com/react/llms-patterns.txt` for recipes and patterns.
- Use the official MCP server:

```json
{
  "mcpServers": {
    "heroui-react": {
      "command": "npx",
      "args": ["-y", "@heroui/react-mcp@latest"]
    }
  }
}
```

## Setup Checks

- Requirements: React >= 19.0.0, Tailwind CSS >= v4, `@heroui/react >= 3.0.0`, and `@heroui/styles`.
- Install packages with the target project's package manager.
- CSS import order matters:

```css
@import "tailwindcss";
@import "@heroui/styles";
```

- HeroUI React v3 does not require a provider. Use `I18nProvider` only when locale setup is needed.
- Run `npx heroui-cli@latest doctor` before final handoff when possible.

## Bundled Agent Docs

- `references/official/react/AGENTS.md`: primary official generated index.
- `references/official/react/llms.txt`: compact index.
- `references/official/react/llms-components.txt`: component documentation.
- `references/official/react/llms-patterns.txt`: patterns, recipes, composition, styling, and theming.
- `references/official/react/llms-full.txt`: complete documentation; load only when necessary.
- `references/official/react/manifest.json`: source URL, generated command, bytes, and sha256 metadata.

## Migration From shadcn

- Replace `@/components/ui/*` imports with `@heroui/react` imports.
- Re-check each replacement in the HeroUI MCP server; do not assume shadcn anatomy maps 1:1.
- Avoid carrying over shadcn-only composition names like `CardHeader`, `CardContent`, `SelectTrigger`, or `Sheet` unless HeroUI documentation explicitly provides the same API.
- Replace `Sheet` patterns with HeroUI `Drawer` or `Modal` based on workflow.
- Replace custom toast wrappers with HeroUI `Toast` when suitable.

## Quality Rules

- Prefer HeroUI theme variables and semantic surfaces over raw Tailwind color families.
- Use official HeroUI components for controls, forms, overlays, feedback, navigation, and data display.
- Keep internal tools dense and scannable; avoid landing-page hero composition for dashboards and operational software.
- Verify every visual surface with desktop and mobile screenshots.
